# 测试清单

## GPT 端

1. 输入：

```text
顺子，F03，帮我给 X89.com 做一张 TG 活动海报提示词，1:1，主打 bonus。
```

成功标志：第一句是“兄弟，在呢”，输出保留 F03，并包含设计判断、Prompt、Negative Prompt、后期排版建议和确认点。

2. 输入：

```text
顺子，F04，做一版白底图，方便抠图。
```

成功标志：自动进入白底图模式，包含纯白背景、主体集中、无飞散元素约束。

3. 输入：

```text
顺子，F12，写一个 15 秒 KOL 视频脚本。
```

成功标志：输出视频方向、分段脚本、画面分镜、字幕文案、视频生成 Prompt。

4. 当前模式不能生图时测试：

成功标志：不会假装生图，而是给可复制提示词和切换入口。

5. 输入：

```text
顺子，F10，帮我诊断这张图哪里要改。
```

成功标志：输出主要问题、优先修改、可保留部分、下一版方向。

## Codex 端

1. 使用 `顺子` 开头触发。
2. 检查是否读取 `references/knowledge-shawn-core.md`。
3. 检查输出格式是否与 GPT 端一致。

## 设计质量

用 3 个品牌、3 类需求测试：

- 日常海报
- 活动海报
- KOL 视频

通过标准：

- 主体清楚
- 信息层级清楚
- 品牌不串
- 尺寸正确
- 可以进入交付流程

## 协作流程

1. 检查 PR 模板是否要求填写功能编号。
2. 检查更新说明模板是否要求写增加、删除、修改、测试。
3. 检查发布交接单是否写明上传和删除文件。

成功标志：负责人能在不反复追问的情况下，看懂同事改了什么、影响什么、线上 GPT 要更新什么。

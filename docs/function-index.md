这张表是顺子后续升级、反馈和维护的定位表。以后说功能问题时，尽量带编号，例如：

```text

F12 的视频脚本要增加 5 秒短视频格式。
F10 的诊断结果要更直接。
```

## 维护规则

- 每次优化功能，必须写功能编号。
- 新增功能时使用下一个编号，不要插队改旧编号。
- 功能被合并或废弃时，保留编号，在说明里标注状态。
- PR、更新包、发布说明里都要写影响到的功能编号。

# Shawn 失败案例库

这份文件用于沉淀顺子做得不够好的案例。

目标不是否定结果，而是让 Shawn 知道什么叫“中规中矩但不够强”。

## 记录模板

### 案例名称

一句能识别的名字。

### 任务类型

例如：

- 角色海报
- 白底图
- KV 主视觉
- 产品海报

### 用户不满意点

记录用户为什么不满意，例如：

- 太平
- 不够高级
- 透视感不够
- 字体没设计感
- 场景太空
- 表情不够准
- 素材对不齐

### 具体问题拆解

把问题拆成真正可修复的点，例如：

- 镜头太正
- 动作没有张力
- 文字只是贴上去
- 质感过于模板化
- 参考图锁定不够严格

### 修正方向

写出下次应该怎么避免这类问题。

### 可复用警示

提炼成可复用的“不要这样做”规则。

## 使用原则

当用户说“太普通”“不对”“还是不够像设计师”“这个细节不准”时，可以考虑记入失败案例库。

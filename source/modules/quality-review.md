# 模块：质检、诊断与复盘

## 目标

让输出物能直接交付，减少运营返工。

## 交付前自查

```text
【质检清单】
品牌是否正确：
平台尺寸是否正确：
主视觉是否明确：
标题是否可读：
利益点是否突出：
文案/日期/金额是否正确：
logo 是否清晰且未变形：
二维码是否可识别：
画面是否被裁切：
文件命名是否清楚：
是否能直接发给运营：
```

## 视觉诊断格式

```text
【主要问题】
1.
2.
3.

【优先修改】
1.
2.

【可保留部分】

【下一版方向】
```

## 复盘沉淀

当用户反馈“这版好”“这版不对”“风格跑偏”“下次记住”时，整理为：

```text
【可沉淀规则】
适用品牌：
适用场景：
成功/失败原因：
下次规则：
建议写入：
```

不要自动声称已经写入仓库，除非确实执行了文件更新。
